# Erros

Neste guia, falaremos sobre o que acontece quando algo dá errado enquanto você trabalha com a API. Erros acontecem e, principalmente, serão seus, NÃO nossos. Vejamos alguns códigos de status e tipos de erros que você pode encontrar. {{ className: 'lead'}}

Você pode saber se sua solicitação foi bem-sucedida verificando o código de status ao receber uma resposta da API. Se uma resposta voltar sem sucesso, você pode usar o tipo de erro e a mensagem de erro para descobrir o que deu errado e fazer alguma depuração rudimentar (antes de entrar em contato com o suporte).

<Note>
Antes de entrar em contato com o suporte com um erro, esteja ciente de que 99% de todos
os erros relatados são, na verdade, erros do Usuário. Portanto, verifique cuidadosamente
seu código antes de entrar em contato com o Suporte ao protocolo.
</Note>

---

## Códigos de Status

Aqui está uma lista das diferentes categorias de códigos de status retornados pela API do Protocolo. Use - os para entender se uma solicitação foi bem-sucedida.

<Properties>
<Property name="2xx">
Um código de status 2xx indica uma resposta bem-sucedida.
</Property>
<Property name="4xx">
Um código de status 4xx indica um erro do cliente-isso significa que é um _Você_
problema.
</Property>
<Property name="5xx">
Um código de status 5xx indica um erro de servidor-você não os verá.
</Property>
</Properties>

---

## Tipos de erro

<Row>
<Col>

Sempre que uma solicitação não for bem-sucedida, a API do Protocolo retornará uma resposta de erro com um tipo de erro e uma mensagem. Você pode usar essas informações para entender melhor o que deu errado e como corrigi-lo. A maioria das mensagens de erro são bastante úteis e acionáveis.

Aqui está uma lista dos dois tipos de erro suportados pela API de Protocolo — use-os para entender o que você fez de errado.

<Properties>
<Property name="api_error">
Isso significa que cometemos um erro, o que é altamente especulativo e improvável.
</Property>
<Property name="invalid_request">
Isso significa que você cometeu um erro, o que é muito mais provável.
</Property>
</Properties>

</Col>
