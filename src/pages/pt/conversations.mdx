# Conversations

As conversas são uma parte essencial do Protocolo — são os recipientes para as mensagens entre você, seus contatos e grupos. Nesta página, vamos nos aprofundar nos diferentes pontos finais de conversação que você pode usar para gerenciar conversas programaticamente. Veremos como consultar, criar, atualizar e Excluir conversas. {{className: 'lead'}}

## The conversation model

O modelo de conversação contém todas as informações sobre as conversas entre si e os seus contactos. Além disso, as conversas também podem ser baseadas em grupo com mais de um contato, podem ter uma mensagem fixada e podem ser silenciadas.

### Properties

<Properties>
  <Property name="id" type="string">
Identificador único da conversa.
  </Property>
  <Property name="contact_id" type="string">
Identificador único para o outro contacto NA conversa.
  </Property>
  <Property name="group_id" type="string">
Identificador único para o grupo ao qual a conversa pertence.
  </Property>
  <Property name="pinned_message_id" type="string">
Identificador único da mensagem fixada.
  </Property>
  <Property name="is_pinned" type="boolean">
Se a conversa foi ou não fixada.
  </Property>
  <Property name="is_muted" type="boolean">
Se a conversa foi ou não silenciada.
  </Property>
  <Property name="last_active_at" type="timestamp">
Carimbo de data / hora de quando a conversa foi ativa pela última vez.
  </Property>
  <Property name="last_opened_at" type="timestamp">
Carimbo de data / hora de quando a conversa foi aberta pela última vez pelo
utilizador.
  </Property>
  <Property name="created_at" type="timestamp">
Data e hora de quando a conversa foi criada.
  </Property>
  <Property name="archived_at" type="timestamp">
Data e hora de quando a conversa foi arquivada.
  </Property>
</Properties>
